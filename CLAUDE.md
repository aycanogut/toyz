## Commands

Package manager is **pnpm** (10.32.1). Use `pnpm` for all installs/scripts.

- `pnpm dev` — Next.js dev server with Turbopack
- `pnpm build` / `pnpm start` — production build / serve
- `pnpm lint` — ESLint (Next + Prettier configs)
- `pnpm test` — Vitest (unit + integration) once
- `pnpm test:watch` / `pnpm test:ui` — Vitest watch / UI dashboard
- `pnpm test:e2e` / `pnpm test:e2e:ui` — Playwright (auto-starts `pnpm dev` on :3000)
- Run a single Vitest file: `pnpm vitest run tests/unit/foo.test.ts` (or `-t "test name"` to filter)
- Run a single Playwright spec: `pnpm playwright test tests/e2e/foo.spec.ts`
- `pnpm generate:env-key-types` — regenerate `types/EnvFileTypes.d.ts` from `.env.*` (run after adding env vars)

## Architecture

This is a **Next.js 15 (App Router) + Payload CMS 3** monorepo-style app where Payload runs in-process inside the same Next app.

### Two route groups under `app/`

- `app/[locale]/` — public site, wrapped by `next-intl` middleware. Locales: `en`, `tr` (default `en`). Routing helpers come from `i18n/routing.ts` — import `Link`, `redirect`, `usePathname`, `useRouter` from there, **not** from `next/navigation`.
- `app/(payload)/` — Payload CMS code: `collections/`, `globals/`, `blocks/`, `fields/`, `jobs/`, plus the admin UI mounted at `/toyz-panel` (overridden via `routes.admin` in `payload.config.ts`, not `/admin`). The middleware matcher in `middleware.ts` excludes `toyz-panel`, `api`, etc. from i18n.

### Payload data model

`payload.config.ts` is the single source of truth. Collections: `Articles`, `Media`, `Categories`, `Events`, `EventMedia`, `Subscribers`. Globals: `Home`, `Slider`, `About`, `Contact`, `EventsGlobal`. Localization is enabled at the Payload level (`en`, `tr`) — most content fields are localized.

- Generated types live in `payload-types.ts` (regenerated by Payload from collection/global schemas — do not hand-edit).
- Media is stored in **Cloudflare R2** via `@payloadcms/storage-s3` (region `auto`); public URLs are rewritten to `https://cdn.toyzwebzine.com/<filename>`. Local storage is disabled.
- Search uses `@payloadcms/plugin-search` over the `articles` collection.
- A rich-text Lexical editor adds a custom `YouTubeBlock` (`app/(payload)/blocks/youtube`).

### Background jobs

Payload's queue runs **in-process** via `autoRun` cron `*/1 * * * *` (every minute, limit 50). The only registered task is `newArticleEmailTask` (notifies subscribers via Resend, template in `emails/NewArticleEmail.tsx`). On boot, `onInit` in `payload.config.ts` recovers stuck jobs (`processing: true` with no `completedAt`) — keep this behavior in mind when debugging job state.

### Data access pattern

UI components do **not** call Payload directly. Instead:

- `services/*.ts` — server-side data fetchers (one file per resource: `articles`, `article`, `home`, `events`, `search`, `sitemap`, etc.). They use the Payload Local API via `utils/payloadClient.ts`.
- `app/[locale]/**/page.tsx` calls these services and passes data to client components.
- Server actions (e.g. contact form, subscribe) live alongside their feature and call Payload + Resend directly; reCAPTCHA is verified server-side via `utils/verifyReCaptcha.ts`.

### Env vars and config

All env access is centralized in `toyzConfig.ts` — import from there rather than reading `process.env` ad-hoc. Types for env keys are generated into `types/EnvFileTypes.d.ts` by `generatorScripts/envFileTypeBuilder.ts` from the `.env.*` files; rerun `pnpm generate:env-key-types` after adding a variable.

### Path aliases

- `@/*` → repo root (Next + Vitest both configured)
- `@payload-config` → `./payload.config.ts` (used by Payload's Next adapter)

### Testing

- Vitest: `tests/unit/`, `tests/integration/`, `tests/components/` — env `happy-dom`, setup in `tests/setup.ts`, only files matching `tests/**/*.test.{ts,tsx}`.
- Playwright: `tests/e2e/`, Chromium only, `webServer` boots `pnpm dev` automatically.

### Misc

- Tailwind v4 (PostCSS plugin) with `@tailwindcss/typography`.
- Shared UI primitives: `components/` (Badge, Button, Icon, Input, Popover). Layout chrome: `layout/Header`, `layout/Footer`. Page-level components: `app/[locale]/components/`.
- Admin URL is `/toyz-panel` (not `/admin`). README mentions `/admin` but the actual mount is `/toyz-panel`.
